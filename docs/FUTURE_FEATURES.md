## 字幕處理

### 1. 可配置的斷句控制 ⭐

**優先級**: P1（如果用戶反饋需要）

**問題**: WhisperX 傾向生成完整句子，有時字幕段落過長。

**解決方案**: 
- 添加 `MAX_CHARS_PER_SEGMENT` 配置選項
- 實現後處理分割邏輯
- 詳細方案見 [WHISPERX_SEGMENTATION.md](WHISPERX_SEGMENTATION.md)

**實現步驟**:
1. 在 `.env` 添加配置選項
2. 在 `transcription_service.py` 實現 `_split_long_segments()` 方法
3. 在前端添加「字幕長度偏好」選項（可選）

**預估工作量**: 2-4 小時

---

## 性能優化

### 7. GPU 加速支持

**優先級**: P3

**功能**:
- 自動檢測 GPU 可用性
- 使用 CUDA 加速轉錄
- 處理速度提升 5-10 倍

**實現**:
- 修改 `transcription_service.py` 的 device 檢測
- 更新 requirements.txt（CUDA 版本的 PyTorch）
- 添加 GPU 記憶體管理

**注意**: 需要 NVIDIA GPU 和 CUDA 環境

## 進階功能

### 13. 說話者分離（Speaker Diarization）

**優先級**: P3

**功能**:
- 識別不同說話者
- 為每個說話者標記字幕
- 支持多人對話場景

**實現**:
- 使用 WhisperX 的 diarization 功能
- 需要 Hugging Face token（免費）
- 字幕格式添加說話者標籤

**注意**: 需要接受 pyannote 模型的用戶協議
