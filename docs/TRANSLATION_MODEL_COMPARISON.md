# 翻譯模型對比分析：Google Translate vs NLLB-200

## 概述

本文檔分析當前使用的 Google Translate 和替代方案 NLLB-200（No Language Left Behind）模型的優劣勢，以及切換的工作量評估。

---

## 當前方案：Google Translate

### 技術實現

**使用庫**：`deep-translator` (GoogleTranslator)  
**API 類型**：免費的網頁 API（非官方）  
**網絡需求**：需要網絡連接

**代碼示例**：
```python
from deep_translator import GoogleTranslator

translator = GoogleTranslator(source='en', target='zh-TW')
translated = translator.translate("Hello world")
# 輸出：你好世界
```

---

### 優勢 ✅

#### 1. 零成本
- **完全免費**：無需 API 密鑰
- **無使用限制**：沒有官方配額限制
- **無需付費訂閱**：適合個人和小型項目

#### 2. 翻譯質量高
- **訓練數據豐富**：Google 擁有海量翻譯數據
- **持續優化**：Google 不斷改進翻譯模型
- **慣用語處理好**：能理解上下文和慣用語
- **專有名詞準確**：對人名、地名、品牌名翻譯準確

#### 3. 語言支援廣泛
- **支援 100+ 種語言**
- **包含所有主流語言**
- **涵蓋本專案需求**：英文、中文（簡繁）、馬來文

#### 4. 實現簡單
- **一行代碼即可使用**
- **無需模型下載**
- **無需 GPU 加速**
- **部署簡單**

#### 5. 維護成本低
- **無需管理模型文件**
- **無需更新模型**
- **無需擔心兼容性問題**

---

### 劣勢 ❌

#### 1. 網絡依賴
- **必須有網絡連接**：離線無法使用
- **受網絡速度影響**：網速慢會影響處理時間
- **可能被封鎖**：某些地區可能無法訪問

#### 2. 速率限制風險
- **可能觸發速率限制**：大量請求可能被暫時封鎖
- **需要添加延遲**：當前實現每個請求延遲 0.3-0.8 秒
- **批次處理受限**：無法快速處理大量字幕

**當前緩解措施**：
```python
# 隨機延遲避免速率限制
await asyncio.sleep(random.uniform(0.3, 0.8))

# 批次間延遲
await asyncio.sleep(2.0)
```

#### 3. 不穩定性
- **API 可能變更**：Google 可能改變網頁 API
- **服務可能中斷**：依賴 Google 服務可用性
- **無官方支援**：使用的是非官方 API

#### 4. 隱私考量
- **數據發送到 Google**：字幕內容會發送到 Google 服務器
- **無法保證隱私**：不適合處理敏感內容
- **可能被記錄**：Google 可能記錄翻譯請求

#### 5. 無法自定義
- **無法微調模型**：不能針對特定領域優化
- **無法控制翻譯風格**：無法調整正式/非正式程度
- **無法添加專業術語庫**

## 替代方案：NLLB-200 模型

### 技術實現

**開發者**：Meta AI  
**模型類型**：本地運行的神經機器翻譯模型  
**使用庫**：`transformers` (Hugging Face)  
**網絡需求**：僅首次下載模型需要網絡

**代碼示例**：
```python
from transformers import AutoTokenizer, AutoModelForSeq2SeqLM

# 載入模型（首次需要下載）
model_name = "facebook/nllb-200-distilled-600M"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSeq2SeqLM.from_pretrained(model_name)

# 翻譯
inputs = tokenizer("Hello world", return_tensors="pt", src_lang="eng_Latn")
outputs = model.generate(**inputs, forced_bos_token_id=tokenizer.lang_code_to_id["zho_Hant"])
translated = tokenizer.decode(outputs[0], skip_special_tokens=True)
# 輸出：你好世界
```

---

### 優勢 ✅

#### 1. 完全離線運行
- **無需網絡連接**：模型下載後可離線使用
- **不受網速影響**：處理速度穩定
- **無地區限制**：任何地方都能使用

#### 2. 無速率限制
- **無限制翻譯**：想翻譯多少就翻譯多少
- **無需延遲**：可以全速處理
- **批次處理快速**：可以並行處理多個句子

#### 3. 隱私保護
- **數據不離開本地**：所有處理在本地完成
- **適合敏感內容**：可以處理機密字幕
- **無追蹤風險**：不會被記錄

#### 4. 穩定可靠
- **不依賴外部服務**：不會因為 API 變更而失效
- **版本可控**：可以固定模型版本
- **官方支援**：Meta AI 官方維護

#### 5. 可自定義
- **可以微調**：可以針對特定領域訓練
- **可以控制參數**：調整翻譯風格
- **可以添加詞彙表**：強制翻譯特定術語

#### 6. 語言支援超廣
- **支援 200 種語言**：包含許多小語種
- **包含方言**：例如粵語、閩南語等
- **涵蓋本專案需求**：英文、中文、馬來文

---

### 劣勢 ❌

#### 1. 模型體積大
- **600M 模型**：約 2.4GB 磁碟空間
- **1.3B 模型**：約 5GB 磁碟空間
- **3.3B 模型**：約 13GB 磁碟空間
- **首次下載慢**：需要時間下載

#### 2. 記憶體需求高
- **600M 模型**：至少需要 4GB RAM
- **1.3B 模型**：至少需要 8GB RAM
- **3.3B 模型**：至少需要 16GB RAM
- **可能需要 GPU**：CPU 運行較慢

#### 3. 處理速度較慢（CPU）
- **CPU 模式**：比 Google Translate 慢 5-10 倍
- **需要 GPU 加速**：才能達到合理速度
- **批次處理有限**：受記憶體限制

**速度對比（估算）**：
```
100 個句子翻譯時間：
- Google Translate：30-60 秒（含延遲）
- NLLB-200 (CPU)：60-120 秒
- NLLB-200 (GPU)：10-20 秒
```

#### 4. 翻譯質量略低
- **不如 Google Translate**：特別是常見語言對
- **慣用語處理較弱**：可能翻譯得較生硬
- **專有名詞可能錯誤**：需要額外處理

**質量對比（主觀評估）**：
```
英文 → 繁體中文：
- Google Translate：9/10
- NLLB-200：7-8/10

英文 → 馬來文：
- Google Translate：8/10
- NLLB-200：7/10
```

#### 5. 實現複雜度高
- **需要安裝 PyTorch**：增加依賴
- **需要管理模型**：下載、更新、版本控制
- **需要處理語言代碼**：NLLB 使用特殊語言代碼
- **需要優化推理**：批次處理、GPU 加速等

#### 6. 維護成本高
- **需要更新模型**：新版本發布時
- **需要處理兼容性**：PyTorch 版本、CUDA 版本等
- **需要監控性能**：記憶體使用、速度等

## 混合方案：兩者並存

### 方案設計

**允許用戶選擇翻譯引擎**：

**.env 配置**：
```env
# 翻譯引擎選擇：google 或 nllb
TRANSLATION_ENGINE=google

# NLLB 配置（僅當選擇 nllb 時）
NLLB_MODEL_SIZE=600M  # 600M, 1.3B, 3.3B
NLLB_DEVICE=cpu       # cpu, cuda
```

**實現**：
```python
# src/services/translation_service.py
class TranslationServiceFactory:
    @staticmethod
    def create(engine_type: str):
        if engine_type == "google":
            return GoogleTranslationService()
        elif engine_type == "nllb":
            return NLLBTranslationService()
        else:
            raise ValueError(f"Unknown engine: {engine_type}")
```

### 優勢
- ✅ 用戶可以根據需求選擇
- ✅ 保持向後兼容
- ✅ 靈活性高

### 劣勢
- ❌ 維護兩套代碼
- ❌ 測試工作量加倍
- ❌ 文檔更複雜

### 額外工作量
- **+5-8 小時**（工廠模式、配置管理、測試）
