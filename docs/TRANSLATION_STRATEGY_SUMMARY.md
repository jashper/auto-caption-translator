## 🎯 策略原則

### ✅ 做什麼

1. **保持 Google Translate**
   - 翻譯質量最好
   - 實現簡單
   - 維護成本低

2. **事前準備**
   - 監控系統（每月 5 分鐘）
   - NLLB 切換代碼模板（已準備）
   - 用戶教育材料（已完成）

3. **定期評估**
   - 每月記錄指標
   - 每季度深度分析
   - 根據數據決策

4. **快速響應**
   - 出現問題時 1-2 天內切換
   - 執行預先準備的檢查清單
   - 測試驗證後上線

### ❌ 不做什麼

1. **不立即重構**
   - 不投入 20 小時開發 NLLB
   - 當前方案運行良好
   - 沒有迫切需求

2. **不完全無視**
   - 不忽略備選方案
   - 不盲目相信 Google 永遠好用
   - 不等到出問題才準備

3. **不過度設計**
   - 不實現複雜的混合方案（除非必要）
   - 不添加不必要的功能
   - 保持簡單

## 🚀 快速切換流程

### 當出現問題時

**時間線**：1-2 天

1. **環境準備**（1-2 小時）
   - 檢查磁碟空間（需要 5GB）
   - 安裝依賴：`pip install transformers sentencepiece`
   - 下載模型：執行測試腳本

2. **代碼部署**（5 分鐘）
   - 複製預先準備的 `nllb_translation_service.py`
   - 修改 `.env`：`TRANSLATION_ENGINE=nllb`
   - 修改 `job_manager.py`：添加引擎選擇邏輯

3. **測試驗證**（30 分鐘）
   - 執行測試腳本
   - 處理測試影片
   - 驗證翻譯質量

4. **上線部署**（5 分鐘）
   - 重啟服務
   - 監控日誌
   - 通知用戶

5. **回滾計劃**（5 分鐘）
   - 如果有問題，改回 `TRANSLATION_ENGINE=google`
   - 重啟服務
